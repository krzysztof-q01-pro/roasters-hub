Gdzie ten komponent się pojawia?

A

Enrichment-first — karta otwiera się po kliknięciu encji w run

Lista runów → Run detail (skrót parametrów) → kliknij encję → Karta encji (pełna edycja + zdjęcie + zatwierdź). Run detail staje się przeglądem encji, nie propozycji.

Ten sam komponent dostępny też z listy kawiarni/palarni (bez kontekstu runu).

B

Entity-first — karta jako osobna strona encji, enrichment ją zasilą

Lista kawiarni/palarni → Karta encji (zawsze dostępna, enrichment tylko wzbogaca propozycjami). Run detail pokazuje "ile encji zaktualizowało dane" z linkami do kart.

Enrichment = feed danych do kart. Karta żyje niezależnie od enrichmentu.

C

Hybrid — Run detail ma embedded kartę dla każdej encji

Run detail to akordeon: każda encja rozwijana = już Karta encji (inline). Bez przechodzenia na inną stronę. Lista kawiarni/palarni linkuje do tej samej karty jako osobna strona.
